Built for the moment before trust

Decide quickly whether it fits your workflow.

Most competing surfaces lean toward broad security platforms, dashboards, or cloud scans. MCP Preflight stays narrow: one local check before you run a new MCP server or ship an agent workflow.

1

Review the config layer

Start with .vscode/mcp.json and other common MCP config locations. The scanner flags insecure transport, credentials in URLs, broad paths, risky launchers, and env passthrough.

2

Review tools and prompts

Descriptions matter. MCP Preflight checks tool text and prompt resources for prompt injection, instruction bypass language, hidden forwarding, and suspicious side effects.

3

Review the repo context

Manifests, dependency specs, lockfiles, and obvious secret-bearing files all shape whether an MCP setup is predictable enough to trust.
