Privacy

Local by default should mean local by default.

This note explains what MCP Preflight reads, what it stores locally, and what it does not send to a hosted service.

Short version

  • The Lite product is designed to run locally.
  • It does not require an account.
  • It does not require a hosted MCP Preflight service.
  • It scans files in the workspace you point it at.

What it reads

The scanner looks at local files that help it answer one practical question: is this MCP setup safe enough to trust?

  • .vscode/mcp.json and other common MCP config files
  • package.json, pyproject.toml, and lockfile presence
  • tool descriptions and prompt resources
  • obvious secret-bearing files such as .env

What it sends

MCP Preflight is built to work without sending your workspace to a hosted MCP Preflight backend. The current Pro unlock path is also local: a signed license token can be stored on disk and verified on the machine without a hosted login flow.

Local activity

MCP Preflight keeps a small local activity log by default so you can inspect scan counts, blocked Pro surfaces, and local license actions on your own machine. The log is local-only. It does not store workspace contents.

  • Disable activity logging with MCP_PREFLIGHT_DISABLE_ACTIVITY=1
  • Change the activity file path with MCP_PREFLIGHT_ACTIVITY_FILE=/path/to/activity-log.jsonl

Important limits

  • Your editor, extension marketplace, operating system, or package manager may still have their own telemetry or network behavior.
  • Third-party MCP servers you choose to run are outside this privacy note.
  • If future paid update delivery changes this behavior, it should be described clearly and separately.

Why this matters

The product is easier to trust when the behavior is explicit: local scan first, readable output, and no surprise account requirement for the free path.

Privacy matters here because the scanner is meant to be trusted before it is automated.
